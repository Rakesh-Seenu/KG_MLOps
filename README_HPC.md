## 🎯 Project Overview

This project demonstrates how to scale Link Prediction on massive heterogeneous biomedical knowledge graphs (BioBridge/PrimeKG) using High-Performance Computing (HPC) clusters.

Standard GNNs suffer from Out-Of-Memory (OOM) errors on large graphs. This architecture solves the scale problem by leveraging **Subgraph Mini-Batching (LinkNeighborLoader)**, **Distributed Data Parallel (DDP)** on multi-node GPUs, and **Mixed Precision Training**.

## 🏗️ Architecture Stack

1. **Heterogeneous GNNs**: PyTorch Geometric `HeteroGNN` using GraphSAGE and GAT layers dynamically wrapped via `to_hetero()`.
2. **Data Pipeline**: PyTorch Lightning `LightningDataModule` managing massive graph splits and asynchronous `LinkNeighborLoader` sub-graph streaming.
3. **Cluster Orchestration**: SLURM bash scripts for hardware allocation (Multi-Node A100s).
4. **MLOps Tracking**: Weights & Biases for tracing system limits and AUROC/Hits@K metrics remotely.

## ⚙️ GPU Cluster Setup

### Environment Bootstrapping
Running Deep Learning on a cluster requires isolated conda environments for CUDA bindings:
```bash
bash scripts/setup_env.sh
```
This provisions a user-space environment `kg_mlops` with PyTorch matching the cluster's base CUDA version.

---

## 🚀 Running Large-Scale Training (SLURM)

Submit the training job to the cluster scheduler:
```bash
sbatch scripts/slurm_submit.sh
```

**What happens under the hood?**
1. SLURM finds available nodes (e.g., 1 node with 4x A100 GPUs).
2. It sets up Distributed Data Parallel (DDP) environment variables.
3. `srun` launches `train_hpc.py`, which instantiates a multi-GPU Lightning Trainer.
4. Gradients are synchronized across GPUs over NCCL while wandb streams realtime data.
